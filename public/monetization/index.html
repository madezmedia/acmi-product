The protocol

Three slots per entity. That's the whole thing.

Every entity in ACMI — an agent, a human, a project, a thread, a work item — has exactly three slots. No schemas, no migrations, no joins. The reference SDK is @madezmedia/acmi on npm; the MCP server is @madezmedia/acmi-mcp, listed publicly on Smithery.

slot · 01

Profile

who

Identity, configuration, role. Stable. Overwrite or merge. Your agent's name, model, expertise, working directories, do/don't-use-for rules.

slot · 02

Signals

now

Current state. What's open. What's hot. A mutable KV map — mood, priorities, scores, the work item this agent is currently focused on.

slot · 03

Timeline

then

Everything that happened, in order. Append-only event log. Every event carries a correlationId — workflows chain across agents and sessions.

FAQ

The honest answers.

Is the protocol really MIT? Why would I pay $47?
Yes, MIT. You're paying for the working fleet — 10 agent configurations, the dashboard, the cron schedule, the Redis layout, the deploy guide. About 10 hours of work you skip. If you'd rather wire it yourself from the GitHub repo, do that. It's free.
Will this work with my agent framework?
ACMI is framework-agnostic. It's a Redis-backed event log with a three-slot model. Anything that can write to Redis (or use the MCP server) can participate. Tested: Claude Code, Cursor, Cline, Cowork, Claude Desktop, Perplexity, LangGraph, CrewAI, raw Node/Python.
What infrastructure do I need?
An Upstash Redis instance (free tier works for getting started) and any MCP-capable agent host. That's it. No vector DB required, no additional servers.
Do I need to know Redis?
No. The deploy guide doesn't ask you to write Redis commands — you call MCP tools or the npm SDK. If you do know Redis, the timeline is just sorted sets and you can ZRANGE it.
How is this different from Mem0 / Letta / Zep / LangMem?
Those tools give one agent better memory. ACMI is a coordination contract for multiple agents to share a timeline. Run Mem0 next to ACMI if you want vector-based recall — they're complementary, not competitive. Letta is an agent runtime; ACMI is a protocol that any runtime can write to.

60 minutes, Tuesdays 2pm ET in Discord. I build the next thing on the public roadmap — a new agent, a new dashboard view, a new MCP integration. You watch the actual code happen, bugs and all. Recording posted within 24 hours.

The patterns work for any agent framework. The Starter Kit's agent prompts are Claude-flavored, but the protocol itself is model-agnostic. If you're 100% on a different stack, the kit gives you the schema and dashboard — you'll write your own prompts.
